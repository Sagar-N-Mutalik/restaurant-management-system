Architectural Canvas

The code behind The Pattern Pizzeria is structured using the famed Gang of Four (GoF) design patterns. These visual metaphors explain the robust engineering powering your experience.

Facade UI

Facade

Provides a simplified interface to a complex subsystem.

When you submit an order, the Facade handles calculating totals, saving to DB, and alerting the kitchen behind a single, clean function call.

Factory

Interfaces for creating objects while letting subclasses decide instantiation.

Our Item Factory seamlessly dynamically generates Pizzas, Burgers, and Sides without cluttering the core transaction logic.

Decorator

Attaches responsibilities or behaviors to an object dynamically.

When you add Extra Cheese or Bacon, we wrap the base item dynamically, layering on prices and descriptions without creating dedicated database entries for every combo.

Observer

A subscription mechanism to notify objects about event changes.

The Kitchen Dashboard simply listens. When an order is placed, the Subject broadcasts the event, allowing the kitchen to update instantly without manual pings.

Strategy

Encapsulates a family of algorithms, making them interchangeable.

Calculating totals is abstracted into varying algorithms (Normal Price, VIP Discount, Happy Hour), allowing us to swap logic instantly at checkout.

Singleton

Ensures a class has only one instance globally.

Used for our Kitchen State Manager. We guarantee there is only one central brain tracking table statuses to prevent overlapping or corrupted data.
